## 한눈에 보기

X(옛 트위터)에서 외국어 트윗을 볼 때마다 **"번역 보기"** 버튼을 일일이 누르는 게 번거로웠던 분들을 위한 크롬 확장이 등장했습니다. 이름은 **Just Translate Already — for X**. 핵심은 단순합니다. 스크롤만 하면 외국어 글이 알아서 번역됩니다.

특히 눈에 띄는 건 **외부로 데이터를 전혀 보내지 않는다**는 점입니다. MIT 라이선스의 오픈소스이기도 하고요.

## 무엇을 해주나요?

X는 외국어 글에 자동 번역을 걸어주기도 하지만, 그렇지 않은 글에는 "번역 보기" 버튼만 띄워둡니다. 이 확장은 바로 **그 버튼을 사용자 대신 자동으로 클릭**해 줍니다. 스크롤을 내리며 글을 읽기만 하면 번역이 자동으로 펼쳐지는 셈이죠.

## 다른 번역 확장과 뭐가 다른가요?

시중의 번역 확장 대부분은 구글 번역 같은 **제3자 엔진**으로 직접 번역해 글 아래에 덧붙입니다. 이 방식에는 두 가지 아쉬운 점이 있습니다.

- 번역 결과가 X가 보여주는 번역과 달라집니다.
- 글 내용이 **외부 번역 서버로 전송**됩니다.

반면 이 확장은 **X 자체(Grok) 번역을 그대로 활용**합니다. X가 원래 띄우는 버튼을 자동으로 눌러줄 뿐이라, 확장이 바깥으로 내보내는 데이터가 하나도 없습니다.

## 동작 방식

- `content script`가 x.com / twitter.com에서 **MutationObserver**로 번역 버튼을 감지해 클릭합니다.
- 버튼 라벨은 UI 언어마다 다르기 때문에, 영어·한국어·일본어·중국어·스페인어·아랍어·힌디어 등 **약 46개 X 인터페이스 언어의 라벨**을 인식합니다. (번역 대상 글의 언어와는 무관합니다.)
- 한 번 처리한 글은 **트윗 ID로 기억**해 다시 클릭하지 않고, "원문 보기"는 절대 누르지 않아 무한 토글이나 루프가 생기지 않습니다.

## 프라이버시

- 요구하는 권한은 `storage` 하나뿐입니다(on/off 토글 저장용). `host_permissions`는 없습니다.
- **네트워크 요청 0, 외부 서버 0, 데이터 수집 0.** 모든 동작이 브라우저 안에서만 이뤄집니다.

## 솔직한 한계

- X가 번역 버튼을 띄우지 않는 곳에서는 동작하지 않습니다. 특히 **대화 스레드의 답글**에는 X가 번역 UI를 잘 제공하지 않습니다. (확장의 버그가 아니라 X의 동작입니다.)
- 계정에 X 자동 번역이 이미 켜져 있다면 대부분 글은 X가 알아서 번역하므로, 이 확장은 그 틈으로 빠지는 글들을 메워주는 **보완재**에 가깝습니다.

## 설치와 코드

오픈소스(MIT)라 코드 전부가 공개돼 있고, 원격 코드나 난독화가 없습니다. 직접 검토한 뒤 `chrome://extensions → 개발자 모드 → 압축해제된 확장 프로그램 로드`로 그대로 설치할 수 있습니다. **별도 빌드 과정도 필요 없습니다.**
