## 한눈에 보는 핵심

- AI 코딩 에디터로 유명한 **Cursor**가 자체 Git 서비스 **Origin**을 **올가을 출시**할 예정입니다.
- 현재는 정식 출시 전 **대기자 명단(Waitlist)** 을 받고 있습니다.
- 신청은 **회사 도메인 이메일로만** 가능해 보이며, gmail 같은 개인 이메일은 등록이 안 되는 것으로 보입니다.

## Cursor가 Git 서비스까지 만든다고?

AI 페어 프로그래밍 도구로 빠르게 입지를 굳힌 Cursor가 이번엔 코드 저장소 영역으로 발을 넓힙니다. 새로 공개된 **Origin**은 Cursor가 준비 중인 Git 서비스로, 올해 가을 출시를 목표로 하고 있습니다.

그동안 Cursor는 GitHub, GitLab 같은 기존 Git 호스팅 위에서 동작하는 에디터였습니다. 여기에 자체 Git 서비스를 더한다는 것은, **코드를 쓰는 단계부터 저장·협업·관리까지** 하나의 흐름으로 묶으려는 시도로 읽힙니다. AI가 코드 변경을 깊게 이해하는 만큼, 저장소 레이어까지 직접 다루면 더 자연스러운 통합이 가능할 거라는 기대가 나옵니다.

## 지금 할 수 있는 건 '대기자 신청'

아직 정식 출시는 아니고, 현재는 **Waitlist 신청**을 받는 단계입니다. 관심이 있다면 미리 이름을 올려두는 게 좋겠습니다. 다만 한 가지 주의할 점이 있습니다.

> 등록은 **회사 도메인을 가진 이메일**로만 가능한 것으로 보입니다. (예: gmail 등 개인 이메일은 등록 불가)

즉, 초기 타깃이 개인 개발자보다는 **팀·조직 단위 사용자**에 맞춰져 있다는 신호로 볼 수 있습니다. 사내 협업 환경에서의 도입을 우선 염두에 둔 전략으로 추측됩니다.

## 정리하며

GitHub가 사실상 표준처럼 자리 잡은 시장에서, AI 코딩 도구가 자체 Git 서비스를 들고 나오는 건 분명 흥미로운 움직임입니다. Origin이 실제로 어떤 차별점을 보여줄지는 가을 출시 이후 확인해봐야겠지만, 회사 이메일 조건이 걸린 점으로 미루어 **B2B·팀 협업** 쪽에 무게를 둔 출발이 될 가능성이 높습니다. 관심 있는 분은 Waitlist 등록부터 챙겨두시길 권합니다.
