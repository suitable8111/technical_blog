## 한눈에 보기

**Paca**는 인간과 AI 에이전트가 *동등한 팀원*으로 함께 일하도록 설계된 오픈소스·셀프 호스팅 프로젝트 관리 플랫폼입니다. Jira, Trello, ClickUp, Monday 같은 상용 협업 도구를 무료로 대체할 수 있는 'AI 네이티브' 대안을 표방합니다. 라이선스는 **Apache License 2.0**.

핵심은 간단합니다. AI가 한쪽 구석의 챗봇 애드온이 아니라, **스프린트에 배정되어 보드 위에 사람과 나란히 표시되는 진짜 팀원**으로 참여한다는 점입니다.

## AI가 'Scrum 팀원'이 된다는 것

기존 도구에서 AI는 보통 질문에 답하는 보조 기능에 머뭅니다. Paca는 여기서 한 걸음 더 나갑니다.

- **통합 Scrumban 보드**에 인간과 AI 에이전트가 함께 표시됩니다.
- 에이전트가 **백로그에서 작업을 직접 가져오고**, 진행 상태를 실시간으로 업데이트합니다.
- **BDD(행위 주도 개발)** 명세를 함께 작성하며 Gherkin 시나리오를 공동으로 만듭니다.
- **시스템 설계 문서(SDD)**에 기여해 아키텍처를 팀 전체가 볼 수 있게 합니다.
- 복잡한 상황을 인간처럼 **탐색·감지·대응**합니다.

## 자연어로 굴러가는 워크플로우

Paca는 **MCP(Model Context Protocol) 서버**를 통해 Claude 같은 MCP 호환 에이전트를 데이터 계층에 직접 연결합니다. 덕분에 이런 식의 자연어 요청이 그대로 처리됩니다.

> "스프린트 3에 OAuth 작업 배정해줘"

특히 **Claude Code의 `/paca` 스킬**을 쓰면 에디터를 벗어나지 않고도 작업·문서·스프린트를 자연어 슬래시 명령으로 관리할 수 있습니다. 개발 흐름을 끊지 않는다는 점이 매력적입니다.

## 보안과 확장성: WASM 샌드박스 + 셀프 호스팅

에이전트에게 권한을 주는 만큼 안전장치도 신경 썼습니다.

- 워크플로우·상태·필드·보드 레이아웃을 코드 없이 **설정 파일**로 구동합니다.
- 플러그인은 **WebAssembly(WASM)**로 컴파일되어, 능력 기반 권한 샌드박스 안에서 실행됩니다.
- 에이전트는 **OpenHands SDK** 기반으로 격리된 샌드박스 컨테이너에서 동작해 호스트 환경에 영향을 주지 않습니다.
- **단일 Docker Compose 명령**으로 셀프 호스팅이 가능하며, 데이터가 자체 인프라를 벗어나지 않습니다.

## 협업을 구조화하는 P-A-C-A 사이클

이름의 유래이기도 한 **P-A-C-A 사이클**(Plan → Act → Check → Adapt)은 협업을 하나의 반복 루프로 구조화합니다. 이 접근은 복잡계 의사결정 모델인 **Cynefin / Stacey 프레임워크**에 근거를 두고 있어, 단순한 작업 관리가 아니라 '복잡성에 어떻게 대응할 것인가'까지 고려한 설계라는 점이 인상적입니다.

## 정리

Paca는 'AI를 도구로 쓴다'에서 'AI와 함께 일한다'로 관점을 옮긴 시도입니다. 셀프 호스팅으로 데이터 주권을 지키면서, MCP·WASM·컨테이너 격리로 안전하게 에이전트를 팀에 합류시키고 싶은 개발팀이라면 충분히 살펴볼 가치가 있습니다. 오픈소스인 만큼 직접 띄워보고 워크플로우에 맞는지 확인해보길 추천합니다.
