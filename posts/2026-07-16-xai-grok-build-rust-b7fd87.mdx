터미널에서 동작하는 AI 코딩 에이전트 **Grok Build**의 소스 코드가 오픈소스로 공개되었습니다. TUI부터 에이전트 런타임까지 전체가 Rust로 작성되어 있어, AI 코딩 도구의 내부 구조가 궁금했던 개발자라면 좋은 학습 자료가 될 것 같습니다. 핵심 내용을 정리해 드릴게요.

## 핵심 요약

- Grok Build를 구성하는 **Rust 소스와 에이전트 런타임**이 GitHub(xai-org)에 공개되었으며, 저장소는 사내 모노레포에서 주기적으로 동기화됩니다.
- 전체 화면 TUI에서 **코드베이스 이해, 파일 편집, 셸 명령 실행, 웹 검색, 장기 작업 관리**를 지원합니다.
- 자체 코드는 **Apache License 2.0**으로 배포되지만, **외부 기여(PR)는 받지 않는** 소스 공개형 프로젝트입니다.

## Grok Build는 어떤 도구인가요?

Grok Build는 명령어 이름 `grok`으로 실행되는 터미널 기반 AI 코딩 에이전트입니다. 실행 방식은 세 가지를 지원합니다.

1. **대화형 모드** — 터미널에서 전체 화면 TUI로 직접 사용
2. **헤드리스 모드** — 스크립트나 CI 파이프라인에서 자동 실행
3. **ACP 연동** — Agent Client Protocol을 통해 편집기(에디터)에 내장

요즘 터미널 에이전트들이 공통적으로 수렴하는 형태(대화형 + 헤드리스 + 에디터 프로토콜)를 그대로 따르고 있다는 점이 흥미롭습니다.

## 설치와 빌드

사전 빌드 바이너리는 macOS, Linux, Windows용으로 제공됩니다.

```bash
curl -fsSL https://x.ai/cli/install.sh | bash   # macOS / Linux / Git Bash
grok --version
```

소스 빌드에는 **Rust와 protoc**이 필요하며, Rust 툴체인 버전은 `rust-toolchain.toml`에 고정되어 있습니다. 공식 빌드 호스트는 macOS와 Linux이고, Windows 소스 빌드는 best-effort로만 지원됩니다.

```bash
cargo run -p xai-grok-pager-bin        # TUI 빌드 후 실행
cargo build -p xai-grok-pager-bin --release
```

참고로 소스 빌드 결과물의 이름은 `xai-grok-pager`이고, 공식 설치판에서 이를 `grok`이라는 이름으로 배포합니다.

## 저장소 구조 살펴보기

코드는 역할별 Rust 크레이트로 잘게 나뉘어 있습니다.

- `xai-grok-pager` — 스크롤백, 프롬프트, 모달, 렌더링 등 **TUI 담당**
- `xai-grok-shell` — **에이전트 런타임**과 leader·stdio·헤드리스 진입점
- `xai-grok-tools` — 터미널, 파일 편집, 검색 등 **도구 구현**
- `xai-grok-workspace` — 호스트 파일 시스템, VCS, 명령 실행, **체크포인트**
- 그 외 설정, MCP, Markdown, 샌드박스 등 보조 크레이트와 벤더링된 서드파티 코드(`third_party/`)

루트 `Cargo.toml`은 자동 생성 파일이라 읽기 전용으로 취급해야 하고, 변경이 필요하면 각 크레이트의 `Cargo.toml`을 수정하는 방식이 권장됩니다. 개발 시에는 전체 워크스페이스 빌드가 느리기 때문에 `cargo check -p <crate>`처럼 **크레이트 단위로 검사**하는 워크플로를 안내하고 있습니다.

## 라이선스: 오픈소스지만 기여는 받지 않음

- 자체 코드는 **Apache License 2.0**, 서드파티·벤더링 코드는 원본 라이선스를 유지합니다.
- **외부 기여는 받지 않는** 구조라, 커뮤니티가 함께 만드는 오픈소스라기보다는 '소스 공개' 모델에 가깝습니다.
- 흥미로운 점은 `THIRD-PARTY-NOTICES`에 **openai/codex와 sst/opencode의 도구 구현 포트**가 명시되어 있다는 것입니다. 경쟁사 오픈소스 에이전트의 도구 구현을 가져와 활용한 흔적을 라이선스 고지에서 확인할 수 있습니다.

## 커뮤니티 반응

공개 직후 Hacker News에서는 벌써 다양한 실험과 파생 프로젝트가 등장했습니다.

- 코드베이스 안에 Mermaid 다이어그램을 유니코드 문자로 그려주는 **독립형 터미널 렌더러**가 발견되었고, 이를 WebAssembly로 컴파일해 브라우저에서 시험해 볼 수 있는 플레이그라운드도 만들어졌습니다.
- 원격 측정(텔레메트리)과 자동 업데이트를 제거한 **프라이버시 포크**, 다중 제공자를 지원하는 CLI, Tauri 기반 데스크톱 GUI 등 파생 프로젝트가 빠르게 늘고 있습니다.
- 다만 "점유율이 낮은 상황에서 꺼낸 **전술적 오픈소스화**"라는 시각과 데이터 프라이버시에 대한 불신 등 냉정한 평가도 함께 나왔습니다. 모델 성능에 대해서도 호평과 혹평이 극명하게 갈리는 분위기입니다.

## 마치며

외부 기여를 받지 않는 형태이긴 하지만, 상용 AI 코딩 에이전트의 TUI·런타임·도구·샌드박스 구현 전체를 Rust 코드로 들여다볼 수 있다는 것만으로도 가치가 큽니다. 터미널 에이전트를 직접 만들어 보고 싶거나 Rust로 대규모 TUI를 어떻게 구성하는지 궁금하다면, 저장소 구조부터 한번 살펴보시길 추천합니다.
