## 한 줄 요약

국가통계포털(KOSIS)이 열어준 MCP 시범 서비스를, 인터넷 연결 없이 노트북에서 돌아가는 9B급 로컬 모델에 붙여본 데모가 공개됐습니다. "합계출산율 보여줘" 한 마디에 통계 검색부터 차트 생성까지 자동으로 처리하고, 파일 저장처럼 위험할 수 있는 작업 앞에서는 사람의 승인을 기다립니다. API 키는 단 한 개도 쓰지 않았습니다.

## 무슨 일이 일어난 걸까요

이 데모의 핵심은 세 가지입니다.

- **로컬 실행**: 모델이 클라우드가 아니라 내 컴퓨터(Ollama)에서 돌아가기 때문에 프롬프트가 외부로 전송되지 않습니다.
- **자동 통계 조회 및 시각화**: "합계출산율 보여줘"라고 말하면 KOSIS에서 관련 통계표를 검색하고, 검증하고, 실데이터를 가져와 area 차트를 그립니다(2015년 1.20명 → 2024년 0.73명으로 하락한 흐름을 바로 확인할 수 있습니다). 같은 통계표로 12개국 비교 로즈 차트, 8개국 히트맵까지 뽑아낼 수 있습니다.
- **HITL(Human-in-the-Loop) 승인**: "요약을 CSV로 저장해줘"처럼 파일을 쓰는 작업은 위험도 L4로 분류되어 승인 창이 뜨고, 사람이 Approve를 눌러야만 실제로 저장됩니다.

1분 21초짜리 한국어 자막 데모 영상도 함께 공개되어 있어 전체 흐름을 눈으로 확인할 수 있습니다.

## 이걸 가능하게 한 프로젝트, Spring AI Playground

이 데모에 쓰인 도구는 **Spring AI Playground**라는 오픈소스로, Spring AI Community가 인큐베이팅 중인 프로젝트입니다.

원래는 Spring AI의 기능들을 하나씩 눌러보는 단순한 플레이그라운드였는데, Spring AI 2.0에서 에이전트와 MCP(Model Context Protocol)가 프레임워크 코어로 흡수되면서 성격이 바뀌었습니다. 지금은 기능 데모용 앱이라기보다, **로컬 LLM 에이전트를 개발하고 테스트하는 워크벤치**에 가깝습니다.

에이전트를 직접 만들어보려면 보통 다음을 전부 코드로 짜야 합니다.

1. MCP 서버 연결
2. 필요한 툴을 찾아 붙이는 로직
3. 위험한 작업에 대한 승인 게이트

Spring AI Playground는 이 과정을 자바 코드 한 줄 없이 데스크톱 앱 UI에서 확인할 수 있게 해줍니다. 기존에 쓰던 MCP 툴에도 MCP 프록시 기능을 통해 HITL 승인 로직을 얹을 수 있고요.

## 눈여겨볼 기능들

- **Dynamic Tool Discovery**: 요청 내용에 맞춰 필요한 도구를 그때그때 검색해서 연결합니다.
- **위험도 기반 승인 (L0~L5)**: 파일 쓰기 같은 위험한 작업은 등급에 따라 승인 창이 뜨고, 승인·거부 처리 흐름을 직접 눈으로 볼 수 있습니다.
- **Observability 대시보드**: 채팅 기록, 툴 호출, 토큰 사용량, 비용, 리스크를 한 화면에서 확인 가능합니다.
- **소스까지 공개**: 자바·스프링으로 에이전트, MCP 클라이언트/서버, HITL을 어떻게 구현했는지 코드로 직접 볼 수 있습니다.

Windows, macOS, Linux용 다운로드가 모두 제공되며 GitHub와 공식 문서에서 더 자세한 내용을 확인할 수 있습니다.

## 참고할 점

KOSIS MCP는 국가데이터처의 시범 서비스이기 때문에 엔드포인트가 향후 변경될 수 있습니다. Spring AI Playground는 이 서비스를 활용하는 독립적인 오픈소스 프로젝트라는 점도 참고하세요.

공공데이터 MCP를 에이전트에 실제로 연결해본 경험이 있거나 HITL 설계에 대한 아이디어가 있다면, 원문 링크를 통해 의견을 남겨보는 것도 좋겠습니다.
