## 한눈에 보기

무려 **1996년에 나온 GIMP 0.54.1**을, 현대 64비트 Linux 시스템에서 그대로 빌드하고 실행할 수 있도록 **Flatpak으로 패키징한 프로젝트**가 등장했습니다. 30년 가까이 묵은 그래픽 편집기를 최신 환경에서 되살리기 위해 어떤 작업이 필요했는지, 그리고 무엇을 기대할 수 있는지 정리해 봤습니다.

## 어떻게 설치하고 실행하나요?

과정은 생각보다 간단합니다.

1. `gimp-0.54.1-8.flatpak` 번들을 내려받습니다.
2. 아래 명령으로 설치합니다.
   ```
   flatpak install --user gimp-0.54.1-8.flatpak
   ```
3. 실행은 다음과 같이 합니다.
   ```
   flatpak run io.github.balooii.gimp054
   ```

실행 환경은 **Wayland에서 xwayland를 통해** 동작합니다. 다만 일부 글꼴을 쓰려면 추가로 **X11 레거시 코어 폰트 패키지**를 설치해야 할 수 있습니다.

## 옛날 GIMP, 지금과는 많이 다릅니다

이 시절의 GIMP는 우리가 아는 GIMP와 꽤 다릅니다.

- **레이어(Layers) 기능이 아예 없습니다.** 여러 이미지를 합치려면 effects나 image ops로 조합하는 방식을 써야 합니다.
- 지원 포맷은 **JPEG, PNG, GIF, TIFF, XPM, TGA**입니다. 단, XPM은 GIMP 0.54에서 저장 기능이 구현되지 않아 **불러오기만** 가능합니다.
- **텍스트 도구**는 host의 X11 core fonts가 X font path에 없으면 OK를 누르는 순간 프로세스가 종료됩니다. 당시 GIMP는 fontconfig/Xft 이전 방식인 `XLoadQueryFont`를 사용하기 때문입니다.

## 설정과 데이터는 어디에?

- 설정 파일 `gimprc`는 (`$XDG_CONFIG_HOME`을 바꾸지 않았다면) `~/.var/app/io.github.balooii.gimp054/config/gimprc`에 위치합니다.
- 첫 실행 후에는 `~/.var/app/io.github.balooii.gimp054/data/plug-ins/`와 `.../data/dev/` 디렉터리가 생성됩니다. 흥미롭게도 **host의 GCC와 C 표준 라이브러리 헤더로 오래된 GIMP 0.54 플러그인 소스를 직접 컴파일**할 수도 있습니다.

## 30년의 간극을 메운 패치들

이 프로젝트의 진짜 알맹이는 호환성 패치입니다. `0001..0011` 패치가 원본 `gimp-0.54.1.fixed.tar.gz`에 적용되며, 다음과 같은 문제들을 다룹니다.

- 현대 toolchain 호환성
- libpng 1.6 접근 방식
- **64비트 LP64 충돌 및 버퍼 오버런** 문제
- Flatpak 경로 적용

라이선스는 패키징 프로젝트가 **0BSD**, GIMP 0.54.1 본체는 **GPL-2.0-or-later**입니다.

## 커뮤니티 반응

Lobste.rs와 Geek 커뮤니티에서는 향수 어린 댓글이 이어졌습니다.

- 같은 해 태어난 Tux 마스코트 이미지가 **수작업 안티앨리어싱**으로 만들어졌다는 점에 놀랐다는 반응
- 초기 GIMP가 한때 닫힌 소스였고, 당시 Linux에서 Motif 기반 도구는 GIMP·nedit·DDD 정도로 손에 꼽혔다는 회고
- Xaw, XView, Tcl/Tk 등 그 시절 위젯 툴킷 선택지의 빈약함이 결국 tkinter, PerlTk 같은 바인딩을 낳았다는 이야기
- "실제로 쓰이는 `.flatpak` 파일을 본 건 처음"이라는 신선한 감상
- 2002~2013년의 떠다니던 자유로운 패널이 그립다는 옛 사용자의 회상

## 마치며

실용성을 위한 프로젝트라기보다는, **소프트웨어 역사를 직접 손으로 만져볼 수 있게 만든 디지털 보존 작업**에 가깝습니다. 레이어도 없고 폰트도 까다롭지만, 1996년의 인터페이스를 오늘 내 화면에서 띄워보는 경험 자체가 이 패키지의 가장 큰 매력일 것입니다.
